Agent KD

Distilling LLM Agent into Small Models with Retrieval and Code Tools

Minki Kang et al.  ·  arXiv:2505.17612  ·  NeurIPS 2025 Spotlight  ·  arxiv.org/abs/2505.17612

논문 요약

대형 언어 모델(LLM)은 복잡한 추론 작업에서 뛰어난 성능을 보이지만 연산 비용이 매우 높아 실제 배포에 제약이 따른다. 기존 접근은 교사 모델의 chain-of-thought(CoT) 궤적을 소형 학생 모델에 모방하게 하는 방식이었는데, 이 방식은 희귀한 사실 지식이나 정밀한 수치 계산이 필요한 상황에서 학생 모델이 환각(hallucination)을 일으키는 문제가 있다.

이 논문은 Agent Distillation 프레임워크를 제안한다. 단순히 CoT 텍스트를 따라 하게 하는 대신, retrieval tool과 code tool을 사용하는 LLM 기반 에이전트의 전체 문제 해결 행동 — 즉 추론(reasoning), 도구 호출(tool call), 관찰(observation), 재추론으로 이어지는 trajectory 전체 — 을 소형 모델로 전이한다.

결과적으로 0.5B, 1.5B, 3B 크기의 소형 모델이 CoT distillation만 한 1.5B, 3B, 7B 모델과 경쟁 가능한 성능을 달성했다. 사실적(factual) 도메인과 수학적(mathematical) 도메인에 걸친 8개 추론 태스크에서 평가했으며, 도메인 내/외 일반화 모두를 포함한다.

핵심 개념 및 방법론

일반 CoT KD와 Agent KD의 차이

일반 CoT KD
질문 → teacher가 사고 과정 텍스트 생성 → student가 그 텍스트를 SFT로 모방
Agent KD (이 논문)
질문 → reasoning → tool call → observation → 다음 reasoning → 최종 답변
student가 이 전체 trajectory(도구 사용 포함)를 학습

두 가지 핵심 기여

  • First-thought prefix: 교사 모델이 trajectory를 생성할 때 첫 번째 사고 단계를 명시적으로 유도하는 프롬프팅 방법. 교사가 생성하는 trajectory의 품질을 높인다.
  • Self-consistent action generation: 테스트 시점에 소형 에이전트의 견고성을 높이는 행동 생성 방식. 동일한 상황에서 일관된 action을 내도록 학습한다.

핵심 통찰

소형 모델이 단순히 최종 답변만 맞히는 게 아니라, retrieval로 사실 정보를 조회하고 code로 계산을 수행하는 tool-using 행동 자체를 체득하면, 파라미터 크기를 한 단계 줄여도 더 큰 CoT 모델과 동등한 성능을 낼 수 있다는 것이 이 논문의 핵심 주장이다.

Structured Agent Distillation for Large Language Model

Jun Liu et al.  ·  arXiv:2505.13820  ·  2025  ·  arxiv.org/abs/2505.13820

논문 요약

LLM 기반 에이전트는 ReAct 프레임워크처럼 추론(reasoning)과 행동(action)을 교차하며 의사결정을 수행한다. 이러한 에이전트를 소형 모델로 압축하는 기존 방법들은 trajectory 전체를 하나의 텍스트 시퀀스로 SFT하거나, 토큰 단위 KL 발산을 최소화하는 방식을 사용한다. 이 접근은 추론 span과 행동 span을 구별하지 않아 각 구성 요소에 맞는 supervision을 제공하지 못한다는 문제가 있다.

이 논문은 Structured Agent Distillation을 제안한다. trajectory를 [REASON] span과 [ACT] span으로 명시적으로 분리하고, 각 span에 서로 다른 손실 함수(segment-specific loss)를 적용하여 교사 모델의 의사결정 구조를 더 정확하게 재현하도록 학습시킨다.

ALFWorld, HotPotQA-ReAct, WebShop 세 벤치마크에서 토큰 단위 distillation 및 단순 imitation learning 기반 baseline보다 일관되게 높은 성능을 보이며, 최소한의 성능 저하로 상당한 모델 압축을 달성했다.

핵심 개념 및 방법론

[REASON]과 [ACT] span 분리의 의미

에이전트의 trajectory는 크게 두 종류의 텍스트로 구성된다.

  • [REASON] span: 다음 행동을 결정하기 위한 내부 사고 과정. 예: "현재 상황을 분석하면 X를 먼저 해야 한다."
  • [ACT] span: 실제 시스템에 전달되는 행동 명령. 예: tool 호출, 결과 JSON 생성, 다음 상태 전환 등.

기존 토큰 수준 KD는 이 두 span을 구별하지 않고 동일한 가중치로 학습하지만, 이 논문은 각 span의 역할과 중요도가 다르다는 점을 활용해 구조 인지적 감독(structure-aware supervision)을 설계한다.

Structured Agent Distillation의 작동 방식
1. 교사 모델의 trajectory에서 [REASON]과 [ACT] span을 태깅
2. [REASON] span: 교사의 추론 충실도(reasoning fidelity)를 유지하는 손실 적용
3. [ACT] span: 교사의 행동 일관성(action consistency)을 유지하는 별도 손실 적용
4. 두 손실을 결합하여 학생 모델 학습

왜 구조적 분리가 중요한가

소형 student 모델이 최종 JSON이나 tool call 결과만 맞히는 게 아니라, 어떤 판단을 거쳐 그 action을 생성했는지의 과정까지 보존해야 실제 agent workflow에서 예상 외 입력에 대응할 수 있다. 추론 span이 무너지면 행동 span도 잘못된 상태에서 결정되기 때문이다.

핵심 기여span-level alignment가 agent 압축에 중요함을 입증

Sub-goal Distillation: A Method to Improve Small Language Agents

Maryam Hashemzadeh et al.  ·  arXiv:2405.02749  ·  2024  ·  arxiv.org/abs/2405.02749

논문 요약

LLM은 장기적으로 이어지는 대화형 작업(long-horizon interactive tasks)에서 막대한 연산 비용과 호출 횟수 제한으로 인해 실용적으로 사용하기 어렵다. 이 논문은 수십억 파라미터 LLM의 성능을 770M 크기의 소형 언어 모델로 전이하는 방법을 제안한다.

핵심 아이디어는 에이전트를 두 모듈로 나누는 계층적 에이전트(hierarchical agent) 구조다. 계획 모듈(planning module)은 LLM으로부터 knowledge distillation을 통해 sub-goal을 생성하는 방법을 학습하고, 실행 모듈(execution module)은 그 sub-goal을 달성하기 위한 기본 행동(elementary action)을 수행한다. 추론 시점에 두 모듈 모두 LLM에 의존하지 않으므로 LLM 호출 비용이 고정 비용(fixed cost)으로 제한된다.

ScienceWorld 벤치마크에서 elementary action만 모방하는 standard imitation learning보다 16.7%(absolute) 향상을 달성했다.

핵심 개념 및 방법론

Sub-goal이란 무엇인가

전체 목표(goal)를 달성하기 위한 중간 단계 목표다. LLM은 oracle path(정답 경로)를 보고 그 경로에 sub-goal 시퀀스를 주석으로 달아준다. 이렇게 주석이 달린 데이터로 두 모듈을 fine-tune한다.

학습 데이터 생성 과정
1. LLM에 oracle path(정답 경로)를 제공
2. LLM이 그 경로에 "sub-goal 시퀀스" 주석을 추가
3. 주석이 달린 데이터로 planning module과 execution module을 각각 fine-tune
추론 시점 구조
사용자 입력 → planning module (sub-goal 생성) → execution module (elementary action 수행)
※ 이 과정에서 LLM 실시간 호출 없음

왜 sub-goal이 elementary action만 모방하는 것보다 나은가

elementary action만 모방하면 개별 행동의 순서를 외우는 수준이 된다. 반면 sub-goal을 학습하면 중간 목표 개념을 이해하고, 그 목표를 달성하기 위한 행동 시퀀스를 계획할 수 있다. 이 계획 능력이 새로운 상황에서의 일반화를 가능하게 한다.

AgentDistill: Training-Free Agent Distillation with Generalizable MCP Boxes

arXiv:2506.14728  ·  2025  ·  arxiv.org/abs/2506.14728

논문 요약

기존 agent distillation 연구들은 대부분 교사 에이전트의 전체 trajectory를 재생하거나 step-by-step 도구 사용을 모방하는 방식을 취한다. 이 방식은 학생 에이전트가 새로운 환경에서 동적으로 계획을 세우고 행동하는 능력을 학습시키기 어렵다는 한계가 있다.

AgentDistill은 추가 학습 없이(training-free) 교사 에이전트가 만든 MCP(Model-Context-Protocols) 모듈을 직접 재사용하는 방식을 제안한다. MCP는 교사 에이전트가 자율적으로 생성한 구조화되고 재사용 가능한 작업 해결 모듈이다. 이 모듈을 학생 에이전트가 재사용함으로써 새로운 문제를 최소한의 감독 없이 해결할 수 있다.

생물의학 및 수학 벤치마크에서 GPT-4o를 활용하는 OctoTools 같은 고성능 시스템과 비견할 성능을 달성했다.

핵심 개념 및 방법론

MCP Box란 무엇인가

MCP(Model-Context-Protocol)는 특정 작업 유형에 대한 해결 절차를 담은 구조화된 모듈이다. 교사 에이전트가 문제를 해결하는 과정에서 자율적으로 이 모듈을 생성하며, 유사한 문제에 재사용할 수 있도록 일반화된 형태로 저장된다.

일반 Agent KD vs AgentDistill
일반 Agent KD: 교사 trajectory로 student weight를 업데이트
AgentDistill: 교사가 만든 MCP 모듈을 student가 직접 재사용 → 학습 없음

왜 이게 가능한가

MCP는 특정 모델에 종속된 지식이 아니라 작업 해결 절차 자체를 담은 모듈이기 때문에, 더 작은 모델도 해당 절차를 따라 실행할 수 있다. 학생 모델의 역할은 MCP가 정의한 절차를 올바르게 실행하는 것이며, 이 절차를 새로 학습할 필요가 없다.

적용 한계

이 방식은 model compression이라기보다 workflow-level knowledge reuse에 가깝다. 모델 자체의 파라미터나 추론 비용이 줄어드는 게 아니라, 교사 에이전트가 사전에 해결한 문제의 솔루션을 재활용하는 것이다.

항목내용
학습 필요 여부없음 (training-free)
벤치마크생물의학, 수학 벤치마크
비교 대상OctoTools (GPT-4o 기반)
특징model compression이 아닌 workflow compression

Chain-of-Agents: End-to-End Agent Foundation Models via Multi-Agent Distillation and Agentic RL

Weizhen Li et al.  ·  arXiv:2508.13167  ·  2025  ·  arxiv.org/abs/2508.13167

논문 요약

기존 multi-agent 시스템은 수동적인 프롬프트 엔지니어링과 복잡한 agent framework에 의존하며, 연산 효율이 낮고 데이터 중심 학습의 이점을 활용하지 못한다는 한계가 있다.

이 논문은 Chain-of-Agents(CoA)라는 새로운 추론 패러다임을 제안한다. 여러 에이전트가 협업하던 작업을 단일 모델이 내부에서 end-to-end로 수행하는 방식이다. 모델은 문제 해결 과정에서 다양한 tool agent와 role-playing agent를 동적으로 활성화해 multi-agent 협업을 단일 모델 내에서 시뮬레이션한다.

학습 방법은 두 단계다. 첫째, 최첨단 multi-agent 시스템의 behavior를 CoA trajectory로 증류해 agentic SFT를 수행한다. 둘째, 검증 가능한 에이전트 태스크에서 agentic RL을 적용해 성능을 추가로 향상시킨다. 이렇게 학습된 모델을 Agent Foundation Model(AFM)이라 부른다.

핵심 개념 및 방법론

Chain-of-Agents 패러다임

기존 방식은 orchestrator가 여러 전문 agent를 순차 호출하는 구조였다. CoA는 이 협업 구조를 단일 모델의 내부 추론 패턴으로 내재화한다. 모델이 한 번의 forward pass 흐름 안에서 여러 역할을 동적으로 수행한다.

학습 파이프라인
단계 1: 최고 성능 multi-agent 시스템 → CoA trajectory 수집 → agentic SFT
단계 2: 검증 가능한 agent 태스크 → agentic RL → AFM

실용적 의미와 비용

이 접근은 여러 번의 LLM 호출 대신 단일 모델 호출로 복잡한 작업을 처리하므로 latency를 낮출 수 있다. 그러나 CoA trajectory 데이터 수집과 agentic RL 학습이 필요하므로 연구 스케일이 크다. 전체 연구 결과(모델 weights, 학습 코드, 데이터)를 오픈소스로 공개했다.

Internalizing Tool Knowledge in Small Language Models via QLoRA Fine-Tuning

Yuval Shemla, Ayal Yakobe, Tanmay Agarwal  ·  arXiv:2605.17774  ·  2026  ·  arxiv.org/html/2605.17774v1

논문 요약

현재 LLM 기반 에이전트 시스템에서는 매 요청마다 전체 tool schema(도구의 이름, 인수 형식, 설명 등)를 prompt에 포함해야 한다. AssetOpsBench 환경에서 이 tool description은 입력 토큰의 82.6%를 차지하며, 이는 심각한 token overhead와 latency 증가를 유발한다.

이 논문은 이 문제를 QLoRA fine-tuning으로 해결한다. 약 1,700개의 tool-use 예시로 소형 모델을 학습시켜, 추론 시점에 tool description 없이도 올바른 structured tool-use plan을 생성할 수 있게 만드는 description-free inference를 실현한다.

Gemma 4 E4B 모델은 AT-F1 0.65, Judge score 3.88을 달성해 full tool description을 제공받는 미세조정 미실시 baseline(AT-F1 0.47, Judge score 2.88)을 능가했다. Qwen3-4B는 Judge score 3.78로 비슷한 성능을 내면서 Gemma 대비 메모리를 62% 줄이고 추론 속도를 2.5배 높였다.

MiniLLM: Knowledge Distillation of Large Language Models

ICLR 2024  ·  openreview.net/forum?id=5h0qf7IBZZ

논문 요약

LLM KD에서 기존 표준 방법인 forward KL divergence는 student가 teacher의 전체 분포를 덮으려 해 확률이 낮은 영역까지 과대 추정하는 문제(coverage-seeking behavior)가 있다. 이로 인해 student가 품질 낮은 텍스트를 생성하는 경향이 강해진다.

MiniLLM은 이를 reverse KL divergence로 대체한다. Reverse KL은 teacher 분포의 주요 모드(mode)에 집중하는 mode-seeking behavior를 유도하여, student가 높은 확률의 정확한 응답에 집중하게 만든다.

GPT-2, OPT, LLaMA 세 모델 계열에서 120M~13B 크기 모델을 대상으로 실험했다. MiniLLM은 SeqKD 대비 GPT-4 점수 기준 약 5%, SFT w/o KD 대비 10%까지 향상됐다. 760M MiniLLM student는 1.5B teacher보다 UnNI 벤치마크에서 더 높은 성능(Rouge-L 36.4 vs 33.1)을 보였다.

핵심 개념 및 방법론

Forward KL vs Reverse KL

구분Forward KLReverse KL (MiniLLM)
최적화 방향KL(teacher ‖ student)KL(student ‖ teacher)
행동 특성coverage-seeking: teacher 분포 전체를 덮으려 함mode-seeking: teacher의 주요 모드에 집중
문제점student가 teacher의 저확률 영역 과대 추정생성 다양성 소폭 감소 (Dist-4: 99.5 → 99.0)

Reverse KL 최적화의 기술적 어려움

텍스트 생성은 이산적(discrete) token 공간에서 자기회귀 샘플링이 필요하므로, gradient를 직접 역전파할 수 없다. 따라서 Policy Gradient 기반 접근이 필요하다. MiniLLM은 안정적 학습을 위해 세 가지 전략을 추가로 도입한다.

  • Single-step decomposition: 긴 시퀀스의 gradient를 단일 단계 예측으로 분해
  • Teacher-mixed sampling: 다음 토큰 샘플링 시 teacher와 student 분포를 혼합하여 reward hacking 방지
  • Length normalization: 모델이 짧고 단순한 응답만 생성해 reverse KL 값을 낮추는 현상(length bias) 방지

Student가 teacher를 능가하는 이유

Teacher 모델은 MLE로 학습되어 exposure bias(학습 시에는 정답 prefix를, 추론 시에는 자기 생성 prefix를 보는 불일치)가 있다. MiniLLM은 policy optimization 방식으로 학습되어 이 bias를 줄이므로, 학생 모델이 일부 평가 지표에서 teacher를 능가할 수 있다.

KD · On-Policy

GKD: On-Policy Distillation of Language Models — Learning from Self-Generated Mistakes

Rishabh Agarwal et al.  ·  arXiv:2306.13649  ·  ICLR 2024  ·  arxiv.org/abs/2306.13649

논문 요약

자기회귀 시퀀스 모델을 위한 기존 KD 방법들은 학습 중 사용하는 출력 시퀀스와 추론 시 student가 실제로 생성하는 시퀀스 사이에 distribution mismatch가 발생한다. 고정된 teacher output만 보고 학습한 student는 자신이 실제로 생성하는 오류 분포를 학습하지 못한다.

GKD(Generalized Knowledge Distillation)는 이 문제를 on-policy distillation으로 해결한다. Student가 학습 중 자기 스스로 시퀀스를 생성하고, 그 생성 결과에 대해 teacher의 feedback을 받아 학습한다. 이를 통해 student는 자신이 실제로 틀리는 상황에서 어떻게 수정해야 하는지를 학습한다.

GKD는 또한 forward KL, reverse KL, JSD 등 다양한 divergence 함수를 사용할 수 있는 유연성을 제공하며, RLHF(RL fine-tuning)와도 통합 가능하다. 요약, 번역, 수학 추론, instruction-tuning 등 다양한 태스크에서 효능이 검증됐다.

핵심 개념 및 방법론

Off-policy vs On-policy KD

구분Off-policy KDOn-policy KD (GKD)
학습 데이터teacher가 생성한 고정 trajectorystudent가 실시간 생성한 trajectory

장점구현 단순, 데이터 재사용 가능student 실제 오류 분포 학습, distribution mismatch 해소
단점student 실제 오류 학습 불가매 iteration student inference 필요 → 학습 비용 증가

Agent에서 On-policy가 중요한 이유

에이전트 workflow에서 student가 한 단계를 잘못 판단하면, 그 결과로 생성된 observation과 이후 모든 action이 달라진다. Off-policy KD는 teacher가 만든 "올바른" trajectory만 보여주므로, student가 자신이 만든 "잘못된" intermediate state에서 어떻게 회복해야 하는지를 학습하지 못한다. On-policy GKD는 이 문제를 직접 다룬다.

GKD의 핵심 알고리즘 (개념)
1. Student가 현재 파라미터로 시퀀스 생성
2. 생성된 시퀀스에 대해 teacher가 log-likelihood 또는 다른 feedback 계산
3. 이 feedback으로 student 파라미터 업데이트
4. 반복

RLHF와의 통합

GKD의 on-policy 구조는 RLHF의 PPO와 유사한 형태다. Teacher를 reward 모델로 간주하면, GKD는 teacher 분포로부터의 KL 제약 하에 student를 최적화하는 RL 문제로 해석할 수 있다. 이를 통해 distillation과 RLHF를 단일 프레임워크로 통합할 수 있다.

PLaD: Preference-based Large Language Model Distillation with Pseudo-Preference Pairs

Rongzhi Zhang et al.  ·  arXiv:2406.02886  ·  ACL 2024 Findings  ·  arxiv.org/abs/2406.02886

논문 요약

LLM KD에서 전통적인 KL divergence 기반 방법은 세 가지 문제에 직면한다. 첫째, 많은 teacher 모델이 API로만 제공되어 internal states(logits)에 접근할 수 없다. 둘째, teacher와 student 간 capacity gap이 커서 student가 teacher 분포를 직접 흉내 내기 어렵다. 셋째, teacher의 잘못된 보정(mis-calibration)이 student에 그대로 전달된다.

PLaD는 이 문제를 pseudo-preference pair 기반의 preference learning으로 우회한다. Teacher output과 student output의 성능 차이를 활용해 "teacher 출력 > student 출력"이라는 preference pair를 자동으로 구성하고, ranking loss로 student의 sequence likelihood 추정을 재보정한다. Student가 teacher를 직접 모방하는 대신, 두 출력 간의 품질 관계를 학습한다.

핵심 개념 및 방법론

Pseudo-preference pair 구성 원리

Teacher와 student 모델 간에는 capacity 차이가 존재하므로, 동일한 prompt에 대해 teacher output이 student output보다 품질이 높다는 전제가 성립한다. 이 전제를 이용해 학습 데이터를 구성한다.

Pseudo-preference pair 구성
prompt: 입력 (예: 사용자 로그 + 기존 memory)
chosen (선호): teacher output (더 좋은 결과)
rejected (비선호): student output (현재 수준의 결과)
→ 이 pair로 ranking loss 학습

Ranking loss가 하는 일

단순히 teacher 답변을 SFT target으로 학습하면, student는 teacher를 모방하는 데 집중한다. Ranking loss는 대신 "chosen이 rejected보다 높은 likelihood를 가져야 한다"는 순서 관계를 학습시킨다. 이는 teacher output 자체를 외우는 게 아니라 품질 순서를 학습하는 방식이다.

세 가지 문제에 대한 해결 방식

  • internal states 접근 불필요: teacher의 logits나 hidden state가 필요 없고, teacher output 텍스트만 있으면 된다.
  • capacity gap 우회: student가 teacher를 완벽히 모방할 수 없어도, teacher output이 student output보다 낫다는 관계만 학습하면 된다.
  • mis-calibration 완화: ranking loss가 절대적 확률 값이 아닌 상대적 순서를 학습하므로, teacher의 잘못된 확률 보정이 덜 전달된다.

KD · Preference Distribution

Capturing Nuanced Preferences: Preference-Aligned Distillation for Small Language Models (PAD)

Yanggan Gu et al.  ·  arXiv:2502.14272  ·  2025  ·  arxiv.org/abs/2502.14272

논문 요약

기존 preference 기반 KD(PLaD 등)는 두 응답을 쌍으로 비교(pairwise comparison)하여 어느 쪽이 더 좋은지만 학습한다. 이 방식은 두 응답 간 차이의 정도(extent of difference)를 무시해, student가 미묘한 선호도(nuanced preference)를 학습하지 못하는 한계가 있다.

PAD(Preference-Aligned Distillation)는 teacher의 preference 지식을 단순 이진 비교가 아닌 모든 잠재적 선호에 대한 확률 분포로 모델링한다. 언어 모델 자체가 reward function으로 기능해 intrinsic preference를 반영할 수 있다는 통찰에 기반한다.

AlpacaEval 2와 Arena-Hard 벤치마크에서 기존 방법 대비 20% 이상 향상됐으며, MT-Bench에서는 Gemma 계열 student가 teacher를 능가하는 결과를 보였다.

PAD의 세 단계

  • 단계 2 — intrinsic preference 구성: teacher와 student 각각에 대해 생성된 응답들의 reward를 계산해 preference 분포를 구성한다. 언어 모델 자체가 자신이 생성한 응답들에 대해 얼마나 선호하는지를 내재적으로 나타낸다.
  • 단계 3 — 분포 일치: student의 intrinsic preference 분포를 teacher의 분포와 일치하도록 학습시킨다.

KD · Preference · KL 결합

Direct Preference Knowledge Distillation for Large Language Models (DPKD)

Yixing Li et al.  ·  arXiv:2406.19774  ·  2024  ·  arxiv.org/abs/2406.19774

논문 요약

전통적인 KL divergence 기반 KD는 LLM 증류에서 두 가지 문제가 있다. 첫째, KL divergence는 두 분포 간의 차이를 측정하는 데 있어 LLM의 복잡한 의미 공간을 충분히 반영하지 못한다. 둘째, 단순 분포 매칭만으로는 "teacher output이 student output보다 더 선호된다"는 상대적 품질 관계를 학습하기 어렵다.

DPKD는 KL 기반 KD와 DPO류 preference learning을 두 단계로 결합한다. 1단계에서 implicit reward와 reverse KL을 함께 최적화하고, 2단계에서 teacher output이 student output보다 선호될 확률을 높이는 preference probability 개선을 수행한다. 120M~13B 크기 모델에서 다양한 데이터셋으로 baseline 대비 응답 정밀도와 exact match 비율 모두에서 성능이 향상됐다.

핵심 개념 및 방법론

Implicit reward function의 역할

언어 모델은 그 자체로 implicit reward function 역할을 할 수 있다. 모델이 특정 시퀀스를 생성할 확률이 높다는 것은 그 시퀀스를 "선호"한다는 의미로 해석할 수 있다. DPKD는 이 implicit reward를 KL divergence의 보완 신호로 활용한다.

DPKD 두 단계 학습
단계 1: implicit reward + reverse KL divergence를 결합한 목적 함수 최적화
→ student가 teacher의 주요 분포 모드를 따르면서 품질 신호도 학습

단계 2: teacher output의 preference probability를 student output보다 높이는 학습
→ "teacher 응답이 student 응답보다 더 좋다"는 관계를 명시적으로 학습

KD와 preference learning의 결합 의의

기존에는 KD(분포 매칭)와 preference learning(DPO, ranking loss)을 별개의 학습 방식으로 취급했다. DPKD는 이 두 방식이 서로 보완적이며 단일 프레임워크로 통합 가능함을 보인다. 특히 implicit reward를 통해 KL과 preference 두 목적 함수를 수학적으로 연결한다.

항목내용
실험 모델 크기120M ~ 13B parameters
평가 지표output response precision, exact match percentage
결과baseline 대비 두 지표 모두 향상

Quantization · Fine-Tuning

QLoRA: Efficient Finetuning of Quantized LLMs

Tim Dettmers et al.  ·  arXiv:2305.14314  ·  NeurIPS 2023  ·  arxiv.org/abs/2305.14314

논문 요약

LLM을 full fine-tuning하면 성능은 높지만 메모리 사용량이 너무 크다. 65B 파라미터 모델은 fp16 기준으로 약 130GB의 VRAM이 필요해 단일 GPU에서는 불가능하다. QLoRA는 이 문제를 해결하기 위해 사전 학습 모델을 4bit로 양자화한 채 고정하고, LoRA adapter로만 gradient를 흘려 fine-tuning하는 방식을 제안한다.

세 가지 핵심 기술(NF4 quantization, double quantization, paged optimizer)을 통해 65B 파라미터 모델을 단일 48GB GPU에서 fine-tuning할 수 있게 했으며, full 16-bit fine-tuning 수준의 성능을 유지한다. Vicuna 벤치마크에서 단일 GPU 24시간 학습으로 ChatGPT 성능의 99.3%에 달하는 Guanaco 모델을 공개했다.

핵심 개념 및 방법론

세 가지 메모리 절감 기술

  • 4-bit NormalFloat (NF4): 정규 분포를 따르는 가중치에 대해 정보 이론적으로 최적화된 4bit 데이터 타입. 기존 INT4보다 양자화 오류가 낮다.
  • Double quantization: 양자화 과정에서 발생하는 quantization constant(스케일 팩터 등)를 다시 양자화해 파라미터당 평균 약 0.5bit를 추가로 절약한다.
  • Paged optimizer: GPU 메모리 스파이크 발생 시 optimizer state를 CPU 메모리로 페이징해 OOM(out-of-memory) 오류를 방지한다.
QLoRA의 학습 구조
Base model: 4bit로 양자화, 가중치 고정 (frozen)
학습 대상: LoRA adapter 파라미터만 (전체의 <1%)
Gradient: frozen 4bit model을 통과해 LoRA adapter로 역전파
배포: 4bit base + adapter 병합 또는 adapter 별도 서빙

QLoRA ≠ KD

QLoRA는 지식 증류 방법이 아니다. KD를 메모리 효율적으로 수행하기 위한 fine-tuning 기법이다. Agent KD 연구에서는 "teacher agent trajectory를 학습 데이터로 사용하고, student를 QLoRA로 학습"하는 방식으로 결합된다.

항목내용
목표65B 모델을 단일 48GB GPU에서 fine-tuning
성능full 16-bit fine-tuning 수준 유지
Guanaco 결과ChatGPT 성능의 99.3% (Vicuna 벤치마크, 24h 단일 GPU)
실험 규모1,000개 이상 모델 fine-tuning, 8개 instruction 데이터셋

Quantization · PTQ · Serving

AWQ: Activation-aware Weight Quantization for On-Device LLM Compression and Acceleration

Ji Lin et al.  ·  MLSys 2024  ·  MLSys 2024 Proceedings

논문 요약

LLM의 거대한 모델 크기는 serving 시 메모리 제약(hardware barrier)과 토큰 생성 속도 저하(memory bandwidth bottleneck)를 유발한다. 기존 low-bit quantization 방법들은 역전파나 재구성(reconstruction) 과정에 의존해 특정 calibration set에 과적합되는 경향이 있다.

AWQ(Activation-aware Weight Quantization)는 activation을 관찰해 중요한 가중치(salient weights) 1%만 보호함으로써 quantization 오류를 크게 줄이는 방식을 제안한다. 역전파나 재구성 없이 per-channel scaling 최적화만 수행하므로 다양한 도메인과 모달리티에 대한 일반화 능력을 보존한다.

데스크탑과 모바일 GPU 모두에서 HuggingFace FP16 구현 대비 3배 이상 speedup을 달성했으며, 모바일 GPU에서 70B LLaMA-2 모델 배포를 최초로 실현했다.

핵심 개념 및 방법론

핵심 관찰: 가중치는 모두 동등하게 중요하지 않다

모델의 가중치 중 특정 1%의 "중요한" 가중치(salient weight)가 전체 성능에 불균형적으로 큰 영향을 미친다. 이 가중치를 낮은 bit로 양자화하면 오류가 크게 발생하므로, 보호할 수 있다면 전체 quantization 오류를 크게 줄일 수 있다.

왜 activation을 보는가

중요한 가중치를 찾는 기준으로 가중치의 크기(magnitude) 대신 활성화(activation)를 사용한다. 가중치가 activation이 큰 채널과 곱해질 때 더 큰 영향을 미치기 때문이다. 활성화가 큰 채널의 가중치를 보호하면 동일한 보호 비율로 더 많은 오류를 방지할 수 있다.

AWQ의 동작 방식
1. calibration 데이터로 각 채널의 activation 크기 측정
2. activation이 큰 채널 = salient weight로 식별
3. per-channel scaling을 최적화해 salient weight를 보호
4. 역전파/재구성 없이 PTQ 수행 → calibration set 과적합 없음

QLoRA vs AWQ의 역할 구분

항목수치

Quantization · PTQ · Second-Order

GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers

Elias Frantar et al.  ·  arXiv:2210.17323  ·  ICLR 2023  ·  arxiv.org/abs/2210.17323

논문 요약

GPT, OPT와 같은 생성형 사전 학습 트랜스포머 모델은 복잡한 언어 모델링에서 탁월한 성능을 보이지만 연산 및 저장 비용이 극도로 높다. 대형 GPT 모델 추론에는 여러 대의 고성능 GPU가 필요해 사용성이 크게 제한된다.

GPTQ는 approximate second-order information에 기반한 one-shot weight quantization 방법이다. 175B 파라미터 GPT 모델을 약 4 GPU 시간 만에 3~4bit로 양자화할 수 있으며, 압축 전 모델 대비 무시할 수 있는 수준의 정확도 저하만 발생한다. 이전 one-shot quantization 방법 대비 압축 이득을 두 배 이상 향상시켰으며, 175B 모델을 단일 GPU에서 실행하는 것을 최초로 실현했다.

핵심 개념 및 방법론

Second-order information 기반 양자화

단순히 가중치 크기(magnitude)로 양자화 오류를 최소화하는 대신, 모델 출력에 대한 각 가중치의 영향을 2차 정보(Hessian 근사)를 통해 추정한다. 이를 통해 어떤 가중치를 낮은 bit로 변환할 때 오류가 가장 적게 발생하는지를 정확하게 계산할 수 있다.

OBQ(Optimal Brain Quantization) 기반

GPTQ는 OBS(Optimal Brain Surgeon) 계열의 알고리즘을 GPT 스케일로 확장한다. 레이어별로 순차적으로 가중치를 양자화하면서, 이미 양자화된 가중치가 만든 오류를 나머지 가중치로 보정(error compensation)하는 방식을 사용한다.

GPTQ의 주요 특징
one-shot: 단일 pass로 전체 모델 양자화 (반복 학습 없음)
layer-wise: 레이어별 독립 최적화로 확장성 확보
approximate 2nd-order: Hessian 역행렬 근사로 계산 비용 감소

극단적 양자화에서의 성능

주로 3~4bit를 목표로 하지만, 2bit 또는 3진(ternary) 양자화에서도 합리적인 정확도를 유지한다는 점을 보였다.

Can Compressed LLMs Truly Act? An Empirical Evaluation of Agentic Capabilities in LLM Compression

Peijie Dong et al.  ·  arXiv:2505.19433  ·  2025  ·  arxiv.org/html/2505.19433v1

논문 요약

기존 LLM 압축 벤치마크들은 perplexity나 GLUE 정확도 같은 단일 턴, 정적 태스크에만 집중해왔다. 실제 에이전트 배포에 필요한 workflow 생성, tool use, 긴 문맥 이해, 실제 환경 적용 능력은 평가하지 않았다.

이 논문은 ACBench(Agent Compression Benchmark)를 제안한다. 4가지 에이전트 능력(action execution, workflow generation, long-context understanding, real-world application)에 걸친 12개 태스크, 15개 모델, quantization(GPTQ, AWQ)과 pruning(Wanda, SparseGPT)을 포괄하는 최초의 agent 중심 압축 벤치마크다.

핵심 경고
reasoning model로 KD된 DeepSeek-R1-Distilled Qwen2.5-7B는 tool use 정확도가 70% → 43.6%로 급락했다.
일반 reasoning 성능이 좋아진 모델이 agent 시나리오에서는 오히려 더 나빠질 수 있다.

핵심 개념 및 방법론

평가한 압축 방법

  • Quantization: GPTQ, AWQ, SmoothQuant
  • Pruning: Magnitude, Wanda, SparseGPT (비구조적 + 2:4 구조적 모두)
  • KD: DeepSeek-R1-Distill 시리즈 (Qwen2.5-7B, LLaMA 기반)

KD가 agent에서 실패하는 두 가지 이유

  • 교사 모델의 한계: DeepSeek-R1 teacher 모델이 학습 시 tool use, 실제 작업 실행 같은 강력한 에이전트 능력을 충분히 갖추지 않았다. 이 능력은 distillation으로 전달될 수 없다.
  • 용량 trade-off: 소형 모델의 제한된 파라미터 공간에서 수학적 추론 같은 핵심 능력을 보존하려면 function calling, tool manipulation 같은 에이전트 능력 보존은 뒤로 밀린다.

권고사항

  • tool use 능력 유지에는 quantization(GPTQ, AWQ)이 pruning보다 효과적이다.
  • hardware support가 있으면 GPTQ 4-bit 또는 FP8을 사용하라.

  • OS, Webshop 같은 구조화 도메인 workflow에는 GPTQ(INT8), AWQ(INT4)를 권장한다.
  • reasoning KD 모델을 agent에 직접 사용하기 전에 반드시 agent-level benchmark로 검증하라.

Agent Cost · Trajectory Reduction

Reducing Cost of LLM Agents with Trajectory Reduction (AgentDiet)

Yuan-An Xiao et al.  ·  arXiv:2509.23586  ·  FSE 2026  ·  arxiv.org/abs/2509.23586

논문 요약

LLM 기반 multi-turn agent 시스템은 소프트웨어 엔지니어링 작업에서 뛰어난 효과를 보이지만, 턴이 거듭될수록 이전 action, observation, tool result가 누적되어 context(trajectory)가 계속 길어진다. 이 누적 trajectory가 생성하는 입력 토큰 연산 비용은 기존 LLM 단일 호출과는 질적으로 다른 비용 구조를 만든다. 그러나 기존 연구와 상용 에이전트 제품은 이 효율성 문제를 대체로 간과해왔다.

이 논문은 기존 agent trajectory를 분석해 useless(쓸모없는), redundant(중복된), expired(만료된) 정보가 광범위하게 퍼져 있음을 확인하고, 이를 실행 중에 자동으로 제거하는 inference-time trajectory reduction 방법인 AgentDiet을 제안한다.

top-performing coding agent에 구현하여 2개 LLM, 2개 벤치마크에서 실험한 결과, 에이전트 성능을 유지하면서 입력 토큰을 39.9~59.7%, 전체 연산 비용을 21.1~35.9% 줄였다. FSE 2026에 채택됐다.

핵심 개념 및 방법론

왜 agent의 비용 구조가 일반 LLM과 다른가

일반 LLM 호출은 하나의 prompt와 output 비용으로 끝난다. 반면 multi-turn agent는 매 step마다 이전 모든 trajectory(action history, observation, tool result)를 context에 누적해 전달한다. 10 step짜리 coding task에서 마지막 step의 input token은 처음 step보다 훨씬 많을 수 있다. 이것이 agent 비용의 주요 원천이다.

Trajectory waste의 세 종류
Useless: 현재 task 목표와 무관한 정보 (verbose log output, irrelevant output 등)
Redundant: 이미 앞 step에서 처리된 정보가 반복 등장
Expired: 이전 step에서는 필요했으나 현재 목표 달성에 더 이상 필요 없는 상태 정보

AgentDiet의 동작 방식

에이전트가 실행되는 도중(inference-time)에 각 step의 trajectory 내에서 위 세 종류의 waste를 식별하고 자동으로 제거한다. 별도의 모델 학습이나 재훈련 없이 동작하는 inference-time 접근이다. 논문 abstract는 "simple yet effective trajectory reduction approach"로만 서술하며, sliding-window 기반 reflection module의 설계 세부사항은 본문 확인 필요 [Needs verification].

ACON: Optimizing Context Compression for Long-horizon LLM Agents

Minki Kang et al.  ·  arXiv:2510.00615  ·  2025  ·  arxiv.org/abs/2510.00615

논문 요약

long-horizon agent 작업은 action history와 environment observation이 계속 쌓이는 구조적 특성 때문에 context가 무한히 늘어난다. 기존 context compression 연구는 단일 단계 작업이나 특정 응용 분야에만 초점을 맞추어 agent의 이 문제를 다루지 못했다.

ACON(Agent Context Optimization)은 observation과 interaction history 양쪽을 concise하고 task-relevant한 형태로 압축하는 통합 프레임워크다. 핵심은 자연어 공간에서 압축 가이드라인을 자동 최적화하는 것으로, "full context로는 성공하지만 compressed context로는 실패하는" trajectory pair를 수집해 LLM이 실패 원인을 분석하고 그 피드백으로 압축 가이드라인을 업데이트한다.

AppWorld, OfficeBench, Multi-objective QA에서 peak token 기준 메모리를 26~54% 줄이면서 task 성능을 대부분 유지하고, 압축 LLM을 소형 모델로 distill했을 때도 정확도의 95% 이상을 보존했다. 소형 LM을 long-horizon agent로 사용할 때 최대 46% 성능 향상도 달성했다.

핵심 개념 및 방법론

일반 context compression과 agent context compression의 차이

일반 text compression은 의미를 보존하면서 길이를 줄이는 것이 목표다. agent context compression은 여기에 더해 task success에 필요한 상태 정보(현재 goal, 이전 실패 원인, 필요한 tool result 등)를 반드시 보존해야 한다. 단순 요약으로는 agent가 잃어버리면 안 되는 정보를 버릴 수 있다.

ACON의 핵심 루프
1. 전체 context → 압축 → compressed context로 agent 실행
2. full context 성공 + compressed context 실패인 trajectory pair 수집
3. capable LLM이 실패 원인 분석 → 압축 가이드라인 업데이트
4. 반복 → 가이드라인 수렴

소형 모델 distillation

최적화된 LLM compressor를 그대로 사용하면 추가 모듈 overhead가 발생한다. 이를 줄이기 위해 최적화된 압축 LLM을 소형 compressor 모델로 distill한다. distill 후에도 정확도의 95% 이상이 유지된다는 점이 핵심이다.

항목수치

소형 compressor distill 후 정확도 유지over 95%
소형 LM agent 성능 향상최대 46%
벤치마크AppWorld, OfficeBench, Multi-objective QA

Less is More: Optimizing Function Calling for LLM Execution on Edge Devices

Varatheepan Paramanayakam et al.  ·  arXiv:2411.15399  ·  DATE 2025  ·  arxiv.org/abs/2411.15399

논문 요약

edge device에서 function-calling agent를 실행할 때 LLM이 너무 많은 tool 목록을 받으면 세 가지 문제가 발생한다: 낮은 task-completion accuracy, 지연 증가, 높은 전력 소비. 현재 LLM들은 edge에서 복잡한 입력이나 다수의 tool을 동시에 효과적으로 관리하지 못한다.

이 논문은 fine-tuning 없이 LLM에 제공하는 tool 후보 수를 동적으로 줄이는 Less-is-More 프레임워크를 제안한다. 핵심 통찰은 "tool 수를 선택적으로 줄이면 function-calling 성능, 실행 시간, 전력 효율이 동시에 향상된다"는 것이다.

BFCL, GeoEngine 벤치마크에서 edge hardware로 실험한 결과, agentic success rate가 향상되고 execution time을 최대 70%, power consumption을 최대 40% 줄였다. DATE 2025에 채택됐다.

핵심 개념 및 방법론

왜 tool 수를 줄이면 성능이 올라가는가

LLM이 매 step마다 전체 tool catalog를 input에 포함하면, (1) schema 텍스트가 input token의 큰 비율을 차지해 실제 쿼리에 집중할 context가 줄어들고, (2) 작은 모델일수록 무관한 tool들 사이에서 올바른 tool을 고르는 능력이 떨어진다. tool 수를 현재 요청에 관련된 것으로 줄이면 두 문제가 동시에 해소된다.

Less-is-More의 세 수준 tool 표현 구조
Level 1 (개별 tool): 각 tool을 개별적으로 검색
Level 2 (tool cluster): 의미상 유사한 tool들을 클러스터로 묶어 검색
Level 3 (전체 tool set): 기존 방식 (모든 tool 제공)
→ 현재 쿼리에 맞는 level을 동적으로 선택해 LLM에 제공

기존 모델 경량화와의 관계

이 논문의 접근은 모델 파라미터를 줄이는 게 아니라 모델이 처리해야 하는 action space(tool 후보)를 줄이는 방식이다. 소형 모델과 결합하면 시너지가 크다: 작은 모델은 tool이 많을 때 정확도가 특히 더 떨어지기 때문에, tool 수 축소가 소형 모델에서 더 큰 효과를 낸다.

SLM · Function Calling · Empirical Study

Small Models, Big Tasks: An Exploratory Empirical Study on Small Language Models for Function Calling

arXiv:2504.19277  ·  2025  ·  arxiv.org/abs/2504.19277

논문 요약

function calling은 올바른 인수를 식별해야 하는 복잡한 작업이다. LLM은 이를 잘 수행하지만 연산 비용이 너무 높다. 소형 언어 모델(SLM)은 edge device에 적합하게 빠르고 가볍지만, function calling에서 실제로 얼마나 쓸 만한지에 대한 체계적인 평가가 부족하다.

이 논문은 zero-shot, few-shot, fine-tuning 세 조건에서, prompt injection 유무를 포함해 다양한 도메인의 function call 생성 능력을 평가하는 탐색적 경험 연구를 수행한다. edge device에서의 latency와 memory usage도 함께 측정한다.

결과: fine-tuning이 가장 좋지만, SLM은 output format(JSON schema, argument validity)을 지키는 데 상당히 고전한다는 한계가 발견됐다. prompt injection에는 대체로 강건하다.

핵심 개념 및 방법론

Format adherence가 function calling의 병목

SLM이 틀린 tool을 고르는 것보다, 올바른 tool을 골랐지만 argument 형식(JSON key 이름, 데이터 타입, required field 누락 등)을 맞추지 못해 실패하는 경우가 많다는 것이 이 논문의 핵심 발견이다. "문장은 그럴듯하게 보이는데 agent 시스템이 파싱 실패"하는 현상의 원인이다.

세 가지 평가 조건 비교
Zero-shot: tool schema만 제공, 예시 없음
Few-shot: tool schema + 예시 몇 개 제공
Fine-tuning: function call 데이터로 학습 → 가장 좋은 성능

DPO training pipeline for agentic tasks

최적화 전략 비교

PEFT (LoRA 등): 파라미터 효율적 fine-tuning
RL-based: 함수 호출 정답 여부를 reward로 학습
DPO: chosen/rejected pair로 preference 학습
Hybrid: SFT + DPO 결합 → 가장 높은 multi-turn 정확도

Peter Belcak et al. (NVIDIA)  ·  arXiv:2506.02153  ·  2025  ·  arxiv.org/abs/2506.02153

논문 요약

이 논문은 실험 결과 중심의 연구 논문이 아니라 position paper다. agent AI 시스템의 구조적 특성을 분석하고, 왜 SLM이 agent 시스템의 많은 호출에서 더 적합한지를 논증한다.

핵심 테제: agent 시스템은 "소수의 특화된 작업을 반복적으로, 거의 변형 없이 수행"하는 형태가 많다. 이런 작업에서 LLM의 범용 대화 능력은 필요하지 않으며, SLM이 더 강력하고(sufficiently powerful), 본질적으로 더 적합하며(inherently more suitable), 경제적으로 더 유리하다(necessarily more economical).

범용 대화 능력이 필수인 경우에는 heterogeneous agentic system(여러 모델을 호출하는 구조)이 자연스러운 해답이며, LLM-to-SLM agent conversion algorithm을 제안한다.

핵심 개념 및 방법론

LLM-to-SLM agent conversion algorithm

현재 LLM으로 운영 중인 agent workflow를 SLM으로 전환하는 일반적 절차를 제안한다. 이는 단순히 모델을 교체하는 게 아니라, 각 LLM 호출 지점을 분석해 (1) SLM으로 대체 가능한 호출과 (2) LLM 유지가 필요한 호출을 구분하고, 전자를 SLM으로 교체하는 체계적 방법론이다.

SLM이 유리한 agent 호출 유형
- 고정된 schema 안에서 structured output 생성 (JSON, function call)
- 반복적이고 변형이 적은 분류/라우팅 판단
- 특정 tool catalog 내에서의 tool 선택
- 사전 정의된 category tree에서의 매핑
LLM fallback이 여전히 필요한 경우
- 개방형 추론 (open-domain reasoning)
- 새로운 category 생성, 예외 처리
- 불확실한 병합, 충돌하는 선호 해석
- 범용 대화 능력이 필수인 사용자 상호작용

경제적 의미

LLM에서 SLM으로의 부분적 전환만으로도 AI agent 산업 전반의 운영/경제적 영향이 엄청날 것이라고 논문은 강조한다. 동일 작업을 10~100배 낮은 토큰 비용으로 수행할 수 있기 때문이다.

핵심 개념 및 방법론

SLM-default + LLM-fallback 아키텍처

기본적으로 SLM을 사용하고 불확실한 경우에만 LLM으로 넘기는 구조를 공식화한다. uncertainty-aware routing과 verifier cascade를 통합해 신뢰할 수 있는 에이전트를 설계한다.

Guided decoding의 역할

XGrammar, Outlines 같은 guided decoding 라이브러리를 사용하면 strict JSON Schema 출력을 강제할 수 있다. 이는 SLM의 format adherence 문제를 inference 레벨에서 해결하는 방식이다. SLM의 언어 생성 능력의 한계를 decoding 제약으로 보완하는 것이다.

SLM 설계 패턴

  • LoRA/QLoRA를 통한 경량 적응: domain-specific tool catalog에 맞춤화

AgentTuning: Enabling Generalized Agent Abilities for LLMs

Aohan Zeng et al. (Tsinghua)  ·  arXiv:2310.12823  ·  2023  ·  arxiv.org/abs/2310.12823

논문 요약

오픈소스 LLM은 다양한 NLP 태스크에서는 뛰어나지만, agent 역할(planning, memorization, tool utilization)에서는 ChatGPT/GPT-4 같은 상용 모델에 비해 "far inferior"한 성능을 보인다. 기존 연구들은 특정 agent 태스크를 위해 프롬프트를 개선하는 데 집중했을 뿐, LLM 자체의 agent 능력을 일반적으로 향상시키면서 general ability를 유지하는 방법은 연구가 부족했다.

AgentTuning은 이 문제를 해결하기 위해 고품질 interaction trajectory를 담은 경량 instruction-tuning 데이터셋인 AgentInstruct를 구성하고, agent 데이터와 일반 instruction 데이터를 혼합하는 hybrid instruction-tuning 전략을 제안한다. Llama 2 계열에 적용해 AgentLM-7B/13B/70B를 만들었으며, AgentLM-70B는 학습 중 보지 않은 agent task에서 GPT-3.5-turbo와 comparable한 성능을 달성했다. General ability는 손상 없이 유지됐다.

핵심 개념 및 방법론

왜 일반 instruction data만으로는 부족한가

일반 NLP 태스크는 단일 turn input → output 형태가 많다. 반면 agent task는 환경 피드백을 받아 다음 action을 결정하는 multi-turn interaction 구조다. 이 trajectory 형태의 데이터가 없으면 모델이 agent 역할을 수행하는 방법을 학습하지 못한다.

Hybrid Instruction-Tuning 전략
AgentInstruct: 고품질 interaction trajectory (agent task 데이터)
+ General instruction data: 일반 도메인 오픈소스 instruction 데이터
→ 두 가지를 혼합하여 학습 → agent ability 향상 + general ability 유지

혼합이 필요한 이유

agent data만으로 학습하면 모델이 agent 형식에 과적합되어 일반 언어 능력이 저하된다(catastrophic forgetting). general data를 섞으면 두 능력을 균형 있게 유지할 수 있다.

네 프로젝트에서의 역할

Qwen3-235B → 10B student 경량화에서 "왜 일반 SFT만으로는 부족한가"의 근거다. profile update trajectory(로그 분석 → signal 추출 → merge 판단 → action 결정)는 agent interaction trajectory이므로, agent instruction data 형태로 만들어야 한다. 또한 agent data 비율이 너무 높으면 일반 instruction-following 능력이 저하될 수 있으므로 혼합이 필요하다.

Agent Tuning · Data Design · Negative Samples

Agent-FLAN: Designing Data and Methods of Effective Agent Tuning for Large Language Models

Zehui Chen et al.  ·  arXiv:2403.12881  ·  2024  ·  arxiv.org/abs/2403.12881

논문 요약

기존 agent training corpus에는 세 가지 구조적 문제가 있다. 첫째, format following과 agent reasoning이 얽혀 있어(entangled) 사전 학습 분포에서 크게 벗어난다. 둘째, LLM이 agent task에 필요한 여러 능력(retrieval, instruction following, reasoning, understanding)을 서로 다른 학습 속도로 습득한다. 셋째, 기존 접근 방식은 agent 능력을 향상시키는 과정에서 hallucination을 유발하는 부작용이 생긴다.

Agent-FLAN은 이 세 문제를 해결하기 위해 (1) training corpus를 능력별로 분해 및 재설계하고, (2) hallucination 억제를 위한 negative sample을 포괄적으로 구성한다. Llama2-7B에 적용해 다양한 agent 평가 데이터셋에서 기존 최고 성능 대비 3.5% 향상을 달성했고, 모델 크기를 늘릴수록 일관되게 성능이 향상됐으며 general ability도 소폭 개선됐다.

문제 1: Format following과 reasoning의 entanglement

기존 agent 학습 데이터는 "ReAct 형식으로 쓰는 방법"과 "어떤 tool을 왜 써야 하는지 추론"이 하나의 데이터에 뒤섞여 있다. 모델 입장에서는 format을 외워야 하는 학습과 reasoning을 해야 하는 학습이 동시에 일어나 혼선이 생긴다. Agent-FLAN은 자연 대화 형식으로 데이터를 정렬(align)해 pretraining 분포에서의 이탈을 줄인다.

능력별 학습 속도 차이 (논문 Figure 3 기준)
Retrieval: 빠르게 수렴
Instruction following: 중간 속도
Reasoning: 느리게 수렴
Understanding: 중간 속도
→ 하나의 데이터셋에 섞으면 빠른 능력이 먼저 과적합되고 느린 능력은 충분히 학습되지 않음

문제 3: Hallucination 억제를 위한 negative sample

agent tuning 중 나타나는 두 종류의 hallucination을 식별한다.

  • Format hallucination: tool call 형식 자체를 잘못 생성 (잘못된 JSON key, 없는 argument 등)
  • Action hallucination: 실행해서는 안 되는 action을 실행하거나, 근거 없는 tool을 호출

이를 억제하기 위해 "이렇게 하면 안 된다"는 negative sample을 포괄적으로 구성하고 학습 데이터에 포함시킨다.
